# Практикум SDET:
## _Задание UI_

Реализовн паралелльный запуск тестов через pytest-xdist
```sh
pytest -n <количество процессов>
```
Также добавлена возмможность запуска через CI/CD GitHub Actions (/.github/workflows/test.yml)

## Тест-кейсы:

1. Самопроверка загрузки ресурса и преобразования данных согласно правилу
2. Создание клиента (Add Customer).
3. Сортировка клиентов по имени (First Name).
4. Удаление клиента.

## Самопроверка преобразования данных согласно правилу
1. Открыть браузер
2. Перейти по ссылке https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager
3. Нажать на кнопку "Add Customer"
4. Ввести эталонный POST_CODE
5. Ввести First Name
6. Ввести Last Name
7. Проверить, что сгенерированное имя соответствует правильному

## Создание клиента (Add Customer). E2E 
1. Открыть браузер
2. Перейти по ссылке https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager
3. Нажать на кнопку "Add Customer"
4. Ввести сгенерированный First Name
5. Ввести Last Name
6. Ввести сгенерированный Post Code
7. Нажать кнопку Add Customer
8. Нажать на кнопку "Customers"
9. Проверить через строку введя имя, что новый пользователь присутствует в таблице(все введенные данные соответствуют)

## Сортировка клиентов по имени (First Name). E2E
1. Открыть браузер
2. Перейти по ссылке https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager
3. Нажать на кнопку "Add Customer"
4. Добавить 4 новых пользователей (Ввести First Name, ввести Last Name, ввести Post Code и нажать кнопку Add Customer)
8. Нажать на кнопку "Customers"
9. Нажать на ссылку First Name для сортировки
10. Убедиться, что пользователи отсортированы по имени по убыванию
11. Нажать на ссылку First Name для сортировки
12. Убедиться, что пользователи отсортированы по имени по возрастанию

## Удаление клиента. E2E
1. Открыть браузер
2. Перейти по ссылке https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager
3. Нажать на кнопку "Add Customer"
4. Ввести сгенерированный First Name
5. Ввести Last Name
6. Ввести сгенерированный Post Code
7. Нажать кнопку Add Customer
8. Нажать на кнопку "Customers"
9. Проверить через строку введя имя, что новый пользователь присутствует в таблице(все введенные данные соответствуют)
10. Удалить этого пользователя
11. Проверить, что новый пользователь удалён(пользователь с введеными данными отсутствует)
